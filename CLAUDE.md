## What This Is

A Doom Emacs configuration repository (github.com/akkerman/dot-doom). This is not a buildable project but an Emacs configuration managed by Doom.

## Doom Commands

```bash
doom sync      # Sync configuration after changes to init.el or packages.el
doom reload    # Reload configuration without restarting Emacs
doom doctor    # Diagnose issues
doom upgrade   # Update Doom and packages
```

## File Structure

- **init.el**: Module declarations - enable/disable Doom modules with flags
- **config.el**: User configuration - settings, keybindings, custom functions
- **packages.el**: Additional package declarations beyond Doom modules
- **custom.el**: Auto-generated by Emacs customize system (don't edit manually)
- **+functions.el**: Custom functions with `viewsource/` namespace prefix
- **snippets/**: Yasnippet templates organized by major mode
- **templates/**: Org-capture and org-roam templates

## Configuration Patterns

### Adding a Package
1. Declare in `packages.el`: `(package! package-name)`
2. Configure in `config.el` using `(use-package! ...)` or `(after! ...)`
3. Run `doom sync`

### Module Flags in init.el
Flags modify module behavior: `(lang +flag1 +flag2)`. Common flags:
- `+lsp`: Enable language server protocol
- `+childframe`: Use child frames for popups

### Custom Functions
Located in `+functions.el`, all prefixed with `viewsource/`. Key functions:
- `viewsource/org-roam-node-from-cite`: Create org-roam node from bibliography citation
- `viewsource/org-export-to-gfm`: Export Org to GitHub-flavored Markdown via pandoc
- `viewsource/org-scrot`: Screenshot capture with flameshot

### Keybinding Convention
Uses `map!` macro with `:leader` prefix (SPC in normal mode):
```elisp
(map! :leader
      :desc "Description" "key" #'function-name)
```
